# Common Pitfalls

Symptom-oriented troubleshooting table. Each entry maps an observed bug to its
root cause and the concrete fix.

| Problem                                                       | Root Cause                                                                                                                 | Solution                                                                                                                                                                                                                                    |
| ------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Input rendered transparent with only bottom border            | Input shapes skipped during raw JSON fill extraction                                                                       | Always search raw JSON for ALL `矩形` shapes, extract fills + radius + borders                                                                                                                                                              |
| Design placeholder text in production                         | Text content not compared in Step 6                                                                                        | Extract real text from child text nodes inside containers                                                                                                                                                                                   |
| Screenshot API returns Parse Error                            | Internal HTTP parsing issue                                                                                                | Use cached PNGs in `screenshots/` directory                                                                                                                                                                                                 |
| `npx lanhu-mcp` fails on Windows                              | PowerShell spawn limitation                                                                                                | Use `node node_modules/@star_work/lanhu-mcp/dist/index.js` directly                                                                                                                                                                         |
| Input background not white despite correct variable           | Container `.input-group` missing `background`; page gradient bleeds through                                                | Add `background` to BOTH container and inner input                                                                                                                                                                                          |
| Sibling shapes merged into one container                      | Design shows independent boxes (e.g. code input + send button) but code uses single container with inner divs              | Split into independent HTML elements matching each design shape, each with own border/radius/background. Check ALL pages with the same pattern (SignInOrSignUpPanel, NewPasswordPanel, etc.) — this merge issue tends to recur across pages |
| Read-only field styled as plain text instead of input-group   | Email/phone display uses `<div class="email-display">` with ad-hoc styles instead of `.input-group` container              | If the design shape has the same dimensions (311×48px), border, radius as input fields, use `.input-group.input-group--readonly`. Visual appearance drives class choice, not functional role                                                |
| Spacing breaks between non-adjacent `.input-group`            | CSS `.input-group + .input-group` fails when `.input-row` or other wrappers sit between them                               | Add combinators for ALL sibling pairs. Or use parent-level flex `gap` / `> * + *` for structure-agnostic spacing                                                                                                                            |
| Quasar components override design tokens                      | `q-input` / `q-btn` add conflicting styles                                                                                 | Replace with plain HTML + custom CSS classes                                                                                                                                                                                                |
| Cookie expired                                                | Session timeout                                                                                                            | Re-run `lanhu_login`                                                                                                                                                                                                                        |
| `tokens.md` out of sync with design data                      | `tokens.md` not updated alongside SKILL.md changes                                                                         | Always update `tokens.md` when design token values change                                                                                                                                                                                   |
| Slice `icon-N.png` mis-mapped to wrong semantic role          | Generic numeric filenames from `lanhu_download_slices` do not match imageRef semantics                                     | Use the SHA-256 cross-reference method in [`./slice-mapping.md`](./slice-mapping.md). Visual inspection is a last-resort fallback; on MePage 84fafb58 it produced 7/8 wrong mappings                                                        |
| Card background color wrong after sync                        | Sampled coords pointed at card corner (rounded-edge antialias) instead of center                                           | Sample at the geometric center of the card; re-run `pick-colors.ps1` with adjusted coords if colors look desaturated                                                                                                                        |
| Gradient type misread as linear when raw JSON says `type: 1`  | Skipped the `gradient.type` field and assumed linear by visual inspection of stops                                         | Always decode `gradient.type`: 0=linear, 1=radial, 2=conic. See [`./raw-json.md`](./raw-json.md) → "Gradient type decoding"                                                                                                                 |
| Glow / decorative layer coordinates drift on wide viewports   | Authored against a 375px artboard but rendered into a full-width `q-page`; absolute px positions break on ≠375px viewports | Escalate per Principle 10. Options: pin to a 375px centered canvas, proportionally scale whole artboard, or full-viewport stretch. Never default silently                                                                                   |
| Skill silently picked a default when design was ambiguous     | Agent filled in a responsive strategy, fallback color, or slice mapping without asking                                     | Apply Principle 10: escalate via `ask_user_question` with 2–4 concrete options (see [`./escalation.md`](./escalation.md)). Do NOT write code for the ambiguous region until the user replies                                                |
| `_raw.json` presumed to exist but was never generated         | Agent assumed `lanhu_get_design` writes raw JSON (it does not)                                                             | Run `node page/lanhu-mcp-assets/fetch-raw.mjs <image_id>` to produce `_raw.json`. This is the only supported way to get raw sketch data                                                                                                     |
| i18n text renders as literal key like `LanguagePage.labels.x` | Component uses short path `t('PageName.labels.x')` but real key is nested under `pages.stack.PageName.labels.x`            | Use project convention `i18nSubPath('pages.stack.PageName')` from `src/utils/common.ts`. See other sibling pages (`AboutPage`, `DevicesPage`) for canonical usage                                                                           |
